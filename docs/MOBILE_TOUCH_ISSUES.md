# 移动端触摸交互问题记录

## 当前实现方案

### 交互设计
1. **键盘按钮 (⌨)** → 切换虚拟键盘开/关（无状态按钮）
2. **双击** → 切换滚动模式/选择模式
3. **滚动模式**：
   - 上下滑动 → 页面滚动
   - 左右滑动 → 光标左右移动
4. **选择模式**：
   - 拖动 → 选择文字
   - 两指点击 → 呼出菜单（复制/粘贴/全选）

### 技术限制
- xterm 使用 canvas 渲染，无法使用 iOS 原生文字选择
- 选择功能需要通过 xterm API 实现

## 存在的问题

### 1. 单击终端弹出/收起虚拟键盘
- 问题：手机单击终端时仍然会触发虚拟键盘的弹出/收起
- 期望：键盘只能通过 ⌨ 按钮控制

### 2. 双击切换不稳定
- 问题：双击检测不灵敏，有时无法正确切换模式
- 可能原因：触摸事件处理逻辑冲突

### 3. 选择模式很难选择
- 问题：选择时出现错位，选择区域不准确
- 可能原因：getCellPosition 函数计算不准确，或 selectionStartPos 被意外重置

### 4. 两指点击无法呼出菜单
- 问题：两指点击没有效果
- 可能原因：
  - iOS Safari 对两指触摸事件的处理方式不同
  - touchstart 事件中 e.touches.length 检测可能不准确

## 下一步计划

1. 禁止单击触发键盘，只保留 ⌨ 按钮控制
2. 改进双击检测算法
3. 调试选择模式的坐标计算
4. 测试 iOS Safari 的两指触摸事件行为
