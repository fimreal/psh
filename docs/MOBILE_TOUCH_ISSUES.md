# 移动端触摸交互问题记录

## 当前实现方案

### 交互设计
1. **键盘按钮 (⌨)** → 切换虚拟键盘开/关（无状态按钮）
2. **双击** → 切换滚动模式/选择模式
3. **滚动模式**：
   - 上下滑动 → 页面滚动
   - 左右滑动 → 光标左右移动
4. **选择模式**：
   - 拖动 → 选择文字
   - 两指点击 → 呼出菜单（复制/粘贴/全选）

### 技术限制
- xterm 使用 canvas 渲染，无法使用 iOS 原生文字选择
- 选择功能需要通过 xterm API 实现

## 存在的问题

### ~~1. 单击终端弹出/收起虚拟键盘~~ ✅ 已修复
- 问题：手机单击终端时仍然会触发虚拟键盘的弹出/收起
- 修复：在 touchstart 事件中添加 `e.preventDefault()` 阻止默认行为

### ~~2. 双击切换不稳定~~ ✅ 已修复
- 问题：双击检测不灵敏，有时无法正确切换模式
- 修复：改进检测算法 - 时间窗口从 300ms 增加到 400ms，位置容差从 30px 增加到 50px

### ~~3. 选择模式很难选择~~ ✅ 已修复
- 问题：选择时出现错位，选择区域不准确
- 修复：简化 `getCellPosition` 函数，移除错误的偏移量计算

### ~~4. 两指点击无法呼出菜单~~ ✅ 已修复
- 问题：两指点击没有效果
- 原因：iOS Safari 的 `e.touches.length` 在 touchstart 时可能不准确
- 修复：使用 `Map` 按触摸标识符跟踪所有活跃触摸点

## iOS Safari 中文输入法修复方案

**问题**: iOS Safari 中文输入法无法输入空格、数字、标点符号

**原因**: xterm.js 的 `onData` 回调在 iOS Safari 中文输入法组合输入时，部分字符（空格、数字、标点）不会触发

**解决方案**:
1. 在 textarea 上监听 `input` 事件作为补充
2. 使用时间戳跟踪已发送的数据，避免重复发送
3. `onData` 发送时记录数据和时间戳
4. `input` 事件发送前检查是否已通过 `onData` 发送
